Homework 1 — 2D Transformations · Reference Sheet
Core Concept — What is a 2D Transformation?
An operation that moves, rotates, or scales a point on screen. Three building blocks:

Translation shifts a point by (tx, ty)
Rotation rotates around the origin by angle θ
Scale multiplies coordinates by a factor s
Homogeneous Coordinates
2D points use a 3rd dummy coordinate w=1 so all three transforms become matrix multiplications.

(x, y) becomes (x, y, 1)
Why 3×3 for 2D?
Translation can't be expressed as a plain 2×2 multiply. Adding the w row makes it possible. All three ops then combine into one matrix multiply per vertex.

Column-Major Order — how WebGL/CSS store matrices
WebGL (and this project) store matrices as a 1D array read column by column, not row by row.

On paper the matrix looks like this:
array[0] array[3] array[6] ← row 0 array[1] array[4] array[7] ← row 1 array[2] array[5] array[8] ← row 2
So index = col×3 + row. The translation values tx,ty live at indices [6] and [7] (column 2).
Combined Transform — order matters
The Rule: apply Scale → Rotation → Translation
Transforms are not commutative: A×B ≠ B×A.
If you translate first then rotate, the object orbits the origin instead of spinning in place.

Scale and Rotation merge into one block because c = cos(θ)×s and sn = sin(θ)×s:
[ c−sntx ]
sncty

The Code — GetTransform
function GetTransform(posX, posY, rotation, scale) { let rad = rotation * Math.PI / 180; // degrees → radians let c = Math.cos(rad) * scale; // cosine × scale let sn = Math.sin(rad) * scale; // sine × scale // Column-major: read each column top→bottom return [ c, sn, 0, // column 0 -sn, c, 0, // column 1 posX, posY, 1 // column 2 ← translation here ]; }
The browser CSS function matrix(a,b,c,d,e,f) receives exactly indices [0],[1],[3],[4],[6],[7] — the prof picks them out of your array.
The Code — ApplyTransform (matrix multiplication)
Needed for the 4 propellers: combine the propeller's local transform with the drone's global transform.
rule result element at (row, col) = dot product of row row of trans2 with column col of trans1.
function ApplyTransform(trans1, trans2) { let result = new Array(9); for (let col = 0; col < 3; col++) { for (let row = 0; row < 3; row++) { result[col*3 + row] = // output at (row,col) trans2[0*3+row]*trans1[col*3+0] + trans2[1*3+row]*trans1[col*3+1] + trans2[2*3+row]*trans1[col*3+2]; } } return result; }
In column-major: element at (row,col) lives at index col×3 + row.
The loop accesses trans2 row-wise and trans1 column-wise — exactly the definition of matrix multiplication.
How It All Fits Together
1 Mouse moves → prof updates uav.positionX/Y
2 Prof calls GetTransform(posX, posY, rotation, scale) → your matrix m
3 Prof applies m to the drone body via CSS matrix()
4 For each propeller: GetTransform(offset, r, 1) → local matrix t
5 Prof calls ApplyTransform(t, m) → combined matrix → propeller placed at correct world position
